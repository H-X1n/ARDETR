# DINO v1
尝试该模型的原因：ViT 对算力要求更高，需要更多的训练数据，结果看也仅 CNN 高一点点，并且其训练得到的特征本身并不具备独特的性质

作者想探究一下使用**自监督学习**，而不是监督学习的方式，是否会让 ViT 性能更好。其动机来源于Transformer 在 NLP 领域取得成功的主要因素之一是使用了自监督预训练，例如 BERT 的 Masked Language Modeling 或者 GPT 的 Language Modeling。这些自监督预训练目标使用句子中的单词来创建代理任务 (Pretext Tasks)，能够提供更丰富的学习信号。然而，在图像领域，图像级别的监督训练任务通常会将图像中包含的丰富视觉信息简化，从而导致学习到的特征不够优质。

# DINO v2
尝试该模型的原因：作者设计了一套自动化的数据清洗方法，可以将未整理过的数据变为一个专用的、多样的并且整理过的数据，形成 LVD-142M 数据集。在模型方面，作者换了一个1B的ViT作为基座模型

# DINO v3
尝试该模型的原因：DINOv3旨在通过模型尺寸和训练数据的放大，提强大且通用通用的模型。DINOv3使用了1.7B的图像数据进行训练，把模型加大到了7B。预训练模型的一个关键特性是在保持冻结的同时实现出色的性能，理想情况下达到与专用模型类似的SOTA性能。在这种情况下，一次前向传递可以在多个任务中提供最前沿的结果，从而节省大量的计算资源——这对于实际应用来说是一个重要的优势，特别是在边缘设备上。其次，一个不依赖于元数据的可扩展自监督训练Pipeline解锁了许多科学应用。通过对不同的图像进行预训练，自监督模型可以在大量领域和任务中进行泛化。
